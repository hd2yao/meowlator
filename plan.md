## 1. 项目现状评估

### 1.1 已完成的工作

✅ **架构与基础设施**
- Monorepo 结构完成（apps、services、ml、infra、docs、tools）
- Docker Compose 本地开发环境就绪
- 数据库迁移框架完成（3 个迁移脚本）
- CI/CD 基础框架（Makefile 命令集）

✅ **核心功能**
- 小程序端侧推理引擎（启发式实现）
- API 业务编排与会话管理
- 云端复判服务（哈希 + 先验融合）
- 反馈入库与权重计算
- 模型发布管理（CANDIDATE → GRAY → ACTIVE → ROLLED_BACK）
- 灰度分桶与确定性路由
- 限流、签名验证、白名单控制

✅ **训练与数据流水线**
- MobileNetV3 训练脚本（Oxford-IIIT Pet + 伪标签）
- ONNX 导出（支持 INT8 量化）
- 反馈数据清洗、清单构建、主动学习任务
- 评估集构建、阈值分析、门禁检查

✅ **文档与设计**
- 项目手册（中文）
- API 文档
- 发布 SOP
- 上线指标
- 实现节点记录
- UI/UX 设计规范（HTML + SVG）

⚠️ **前端页面实现**
- UI 设计规范已完成，基础版已实现，需按设计稿重构
- 需要补充：首页、结果页、反馈弹窗的完整 WXML/WXSS/TS 实现

✅ **测试覆盖**
- Go 单元测试（API、推理服务）
- Python 单元测试（训练脚本）
- TypeScript 类型检查（小程序）

### 1.2 当前限制与已知问题

⚠️ **推理实现**
- 端侧与云侧均为启发式/模拟推理，非 ONNX 真推理
- 结果确定性但不代表真实模型能力
- 需要切换到 ONNX Runtime 以获得真实推理能力

⚠️ **训练数据**
- 当前仅使用 Oxford-IIIT Pet（伪标签映射）+ 烟雾测试数据
- 真实用户反馈数据量有限
- 需要积累更多反馈样本以改进模型

⚠️ **部署与运维**
- 本地测试与正式部署存在差异（内存 vs MySQL、HTTP vs HTTPS）
- 监控告警系统未完全建立
- 成本控制策略需要在实际运营中验证

⚠️ **合规与安全**
- 免责声明覆盖率需确认
- 用户数据删除链路需完整测试
- 设备白名单与配额控制需在真实场景中验证

## 2. 短期计划（1-2 周）

### 2.0 前端页面实现

**目标**：完成小程序的三个核心页面实现

**任务**：
1. 首页 (Home Screen) 实现 - P0
   - 页面文件：`pages/index/index.wxml/wxss/ts`
   - 功能：拍照按钮、相册选择、合规提示
   - 状态管理：用户信息、会话状态

2. 结果页 (Result Screen) 实现 - P0
   - 页面文件：`pages/result/result.wxml/wxss/ts`
   - 功能：图片展示、推理结果、文案展示、反馈按钮
   - 状态管理：推理结果、置信度、来源标记

3. 反馈弹窗 (Feedback Modal) 实现 - P0
   - 组件文件：`components/feedback-modal/feedback-modal.wxml/wxss/ts`
   - 功能：意图选择网格、提交反馈
   - 状态管理：选中意图、提交状态

**输出物**：
- 完整的页面与组件代码（P0 功能）
- 样式文件（WXSS）
- 类型定义（TypeScript）
- 页面测试用例

**预计工作量**：3-5 天

**目标**：建立完整的测试体系

**任务**：
1. 单元测试（Jest、Go testing、unittest）
2. 集成测试（API、数据库、前后端）
3. E2E 测试（miniprogram-automator）
4. 性能测试（基准、压力、内存 CPU）

**工具链**：
- 单元测试：Jest（前端）、Go testing（后端）、unittest（Python）
- 集成测试：Postman + Newman（API）、Docker Compose（本地环境）
- E2E 测试：miniprogram-automator（小程序官方工具）
- 性能测试：Apache JMeter（压力测试）、小程序性能评分

**性能基准**（参考 docs/KPI_BASELINE.md）：
- 端侧推理：< 100ms
- API 响应：< 300ms (p95)
- 文案生成：< 5s
- 页面加载：< 2s
- 首屏渲染：< 1s

## 6. 风险评估与应对

### 6.1 技术风险

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| ONNX Runtime 在小程序中不可用 | 中 | 高 | 提前调研 WASM 方案，准备备选方案 |
| 模型推理准确率低于预期 | 中 | 高 | 积累更多训练数据，尝试不同的模型架构 |
| 云端复判服务性能瓶颈 | 低 | 中 | 实现缓存、批处理、模型优化 |
| 数据库性能问题 | 低 | 中 | 实现索引优化、分库分表、读写分离 |
| 前端页面实现延期 | 高 | 高 | 增加前端工程师、简化 MVP 功能、提前启动开发 |
| 前端 UI 质量问题 | 中 | 中 | 建立 UI 测试标准、进行 code review、用户测试 |
| 小程序兼容性问题 | 中 | 中 | 支持 iOS 与 Android 双平台测试、处理版本差异 |
| API 接口不稳定 | 中 | 高 | 加强 API 测试、实现降级策略、监控告警 |
| 数据一致性问题 | 低 | 高 | 实现事务管理、数据校验、审计日志 |

### 6.2 业务风险

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|---------|
| 用户增长缓慢 | 中 | 高 | 优化营销策略，改进用户体验 |
| 用户留存率低 | 中 | 高 | 分析流失原因，实现重新激活活动 |
| 成本超预算 | 中 | 中 | 严格控制资源投入、定期成本审查 |
| 竞争对手出现 | 低 | 中 | 加快功能迭代，建立用户粘性 |

### 6.3 合规风险

| 风险 | 概率 | 影响 | 应对措施 |

## 10. 沟通与协调机制

### 10.1 每日站会

**时间**：每天 10:00-10:15（15 分钟）

**参与人员**：技术负责人、前端工程师、后端工程师、QA、产品经理

**议程**：
1. 昨日完成（2 分钟）
2. 今日计划（2 分钟）
3. 阻塞问题（5 分钟）
4. 风险预警（3 分钟）
5. 下一步行动（3 分钟）

### 10.2 周报与评审

**周报**（每周五 17:00）：本周完成、下周计划、风险问题、成本资源

**周评审**（每周五 17:30）：代码审查、测试结果、性能指标、优先级确认

### 10.3 问题跟踪与升级

**问题分类**：
- P0（阻塞）：影响 M0 完成，需要立即处理
- P1（高）：影响进度，需要 24 小时内处理
- P2（中）：可以延后，需要 1 周内处理
- P3（低）：可选，可以延后到下个版本

**升级流程**：开发者 → 负责人 → 技术负责人 → CTO → 项目经理

### 10.4 变更管理流程

**变更类型**：功能变更、技术变更、时间变更、资源变更

**变更流程**：提出 → 评估 → 批准 → 更新 → 通知

### 10.5 文档与知识管理

**文档清单**：计划、技术设计、测试计划、发布 SOP、故障排查、知识库

**文档工具**：Confluence、GitHub Wiki、Markdown

### 11.1 开发命令

```bash
# 启停
make up
make down

# 测试
make test
make test-go
make test-py
make test-mini

# 训练与导出
make train-vision
make train-vision-smoke
make train-vision-resume
make export-onnx

# 数据流水线
make clean-feedback-data
make build-training-manifest
make active-learning-daily
make build-eval-splits
make threshold-report
make evaluate-intent
make gate-model
```

### 11.2 发布命令

```bash
# 注册候选模型
curl -X POST http://localhost:8080/v1/admin/models/register \
  -H "X-Admin-Token: <token>" \
  -H "Content-Type: application/json" \
  -d '{"modelVersion":"<version>","metrics":{...}}'

# 灰度发布 10%
curl -X POST http://localhost:8080/v1/admin/models/rollout \
  -H "X-Admin-Token: <token>" \
  -H "Content-Type: application/json" \
  -d '{"modelVersion":"<version>","rolloutRatio":0.1,"targetBucket":0}'

# 全量激活
curl -X POST http://localhost:8080/v1/admin/models/activate \
  -H "X-Admin-Token: <token>" \
  -H "Content-Type: application/json" \
  -d '{"modelVersion":"<version>"}'
```
