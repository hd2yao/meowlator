## 1. 性能基准（冻结）

> 说明：目标阈值用于运营达标，告警阈值用于监控触发；两者含义不同，不应混用。

### 推理性能

| 指标 | 目标阈值 | 告警阈值 | 说明 | 测量方法 |
|------|---------|---------|------|---------|
| finalize p95 延迟 | < 500ms | > 2500ms | 推理响应时间 | Prometheus 指标 |
| 错误率 | < 1.5% | > 1.5% | API 错误率 | API 日志统计 |
| 端侧推理 | < 100ms | - | 包括模型加载、前处理、推理、后处理 | 小程序 wx.getPerformance |
| 文案生成 | < 5s | - | 包括 LLM 调用、缓存查询 | 应用日志 |

### 页面性能

| 指标 | 目标 | 说明 | 测量方法 |
|------|------|------|---------|
| 页面加载 | < 2s | 首屏完全加载 | 小程序性能评分 |
| 首屏渲染 | < 1s | 首屏内容可见 | 小程序性能评分 |
| 图片上传 | > 1MB/s | 网络上传速度 | 应用日志 |

### 准确率指标

| 指标 | 目标 | 说明 | 测量方法 |
|------|------|------|---------|
| 推理准确率 | ≥ 85% | 基于用户反馈 | 反馈数据分析 |
| 端侧命中率 | > 70% | 端侧推理直接返回的比例 | API 日志统计 |
| 云端兜底比例 | < 30% | 需要云端复判的比例 | API 日志统计 |

## 3. 上线门禁（M6 ONNX 切换）

### 必须满足的条件（ALL）

| 门禁项 | 条件 | 说明 |
|--------|------|------|
| 准确率 | ≥ 85% | 相对基线，基于用户反馈 |
| 错误率 | < 1.5% | API 错误率 |
| p95 延迟 | < 500ms | 推理响应时间 |
| 单次成本 | < ¥0.2 | 云端复判成本 |
| 云端兜底比例 | < 30% | 端侧命中率 > 70% |
| 测试覆盖率 | > 80% | 单元 + 集成 + E2E |
| 灰度演练 | 成功 | 10% → 30% → 60% → 100% |
| 回滚演练 | 成功 | 能够快速回滚 |

### 回滚触发条件（ANY）

满足任一条件即触发回滚：

| 触发条件 | 阈值 | 说明 |
|---------|------|------|
| 错误率上升 | > 基线 50% | 相对基线上升 50% 以上 |
| 延迟恶化 | > 基线 20% | p95 延迟超过基线 20% 以上 |
| 用户投诉 | > 基线 100% | 投诉量翻倍 |
| 成本超支 | > 预算 30% | 单次成本超过预算 30% |

## 6. 变更流程

### 性能基准变更

**触发条件**：
- 技术方案变更（如 ONNX 集成）
- 硬件升级
- 用户反馈

**变更流程**：
1. 技术负责人提出变更申请
2. CTO 审批
3. 更新此文档
4. 同步到 plan.md 与 research.md
5. 通知所有相关人员

**变更记录**：
```
【变更记录】
日期：2026-02-25
变更项：端侧推理性能基准
原值：< 50ms
新值：< 100ms
原因：保守估计，包括模型加载
审批人：CTO
```

### 成本预算变更

**触发条件**：
- 人员调整
- 基础设施成本变化
- 业务规模变化

**变更流程**：
1. CFO 提出变更申请
2. 业务负责人审批
3. 更新此文档
4. 同步到 plan.md
5. 通知所有相关人员

### 上线门禁变更

**触发条件**：
- 业务需求变化
- 市场竞争压力
- 技术能力提升

**变更流程**：
1. 产品负责人提出变更申请
2. 技术负责人 + 业务负责人审批
3. 更新此文档
4. 同步到 plan.md 与 research.md
5. 通知所有相关人员
